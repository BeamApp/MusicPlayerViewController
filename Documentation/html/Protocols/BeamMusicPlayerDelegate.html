BeamMusicPlayerDelegate Protocol Reference

Conforms to NSObject
Declared in BeamMusicPlayerDelegate.h

Overview

The Delegate of the BeamMusicPlayerViewController must adopt the BeamMusicPlayerDelegate protocol to track changes in the state of the music player.

Tasks

  • – musicPlayerDidStartPlaying:

    Called by the player after the player started playing a song.

musicPlayer:didChangeRepeatMode:

Called when the player changes it’s repeat mode.

- (void)musicPlayer:(BeamMusicPlayerViewController *)player didChangeRepeatMode:(MPMusicRepeatMode)repeatMode

Parameters

player

The BeamMusicPlayerViewController that changes the repeat mode.

repeatMode

a MPMusicRepeatMode indicating the currently active mode.

Discussion

The repeat modes are taken from MediaPlayer framework and indicate whether the player is in No Repeat, Repeat Once or Repeat All mode.

musicPlayer:didChangeShuffleState:

Called when the player changes it’s shuffle state.

- (void)musicPlayer:(BeamMusicPlayerViewController *)player didChangeShuffleState:(BOOL)shuffling

Parameters

player

The BeamMusicPlayerViewController that changes the shuffle state

shuffling

YES if shuffling, NO if not

Discussion

YES indicates the player is shuffling now, i.e. randomly selecting a next track from the valid range of tracks, NO means there is no shuffling.

Called when the player’s volume changed

Parameters

player

The BeamMusicPlayerViewController changing the volume

volume

A float holding the volume on a range from 0.0f to 1.0f

Discussion

Note that this not actually change the volume of anything, but is rather a result of a change in the internal state of the BeamMusicPlayerViewController. If you want to change the volume of a playback module, you can implement this method.

musicPlayer:shouldChangeTrack:

Called before the player actually skips to the next song, but after the user initiated that action.

- (BOOL)musicPlayer:(BeamMusicPlayerViewController *)player shouldChangeTrack:(NSUInteger)track

Parameters

player

the BeamMusicPlayerViewController sending the message

track

a NSUInteger containing the number of the new track

Return Value

YES if the track can be changed, NO if not. Default YES.

Discussion

If an implementation returns NO, the track will not be changed, if it returns YES the track will be changed. If you do not implement this method, YES is assumed.
